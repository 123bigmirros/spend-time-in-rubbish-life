# info collect module design
基于具体问题具体分析的科学原则，这篇倾向于收集能否帮助你解决当前生存困境的一切信息，再为你
提供服务。
但我们也必须考虑到一次性要求提交过多的信息会影响用户体验，阻碍服务推广。


## 系统构思
- 输入：用户个人详细信息
- 输出: 用户画像  
- 难点: 
    1. 在保证用户体验的前提下，收集用户信息。 
    2. 哪些是我们需要收集的信息  
    3. 构建怎样的用户画像  

## 系统设计  
1. 以地球仪的形式获取用户地理位置作为输入信息(对于这个章节的目的而言，地理位置信息是最需要的因素)
2. 利用大模型向用户推荐机会并补充细节信息。
